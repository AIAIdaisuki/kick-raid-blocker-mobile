Kick.com のレイド/ホスト自動リダイレクトをブロック · iPhone・Android・PC 対応

Userscript をインストール

タップで Tampermonkey / Userscripts に取り込まれます

初めての方は先にこちら

  1. Userscripts（無料・App Store）をインストール
  2. iOS 設定 → Safari → 機能拡張 で Userscripts をオン → すべての Web サイト に許可
  3. このページの上の「Userscript をインストール」ボタンをタップ
  4. Userscripts のダイアログで Save
  5. kick.com を開いて、右下の 🛡 ボタンが出れば成功

Kiwi Browser を使う場合（おすすめ）:

  1. Play Store で Kiwi Browser をインストール
  2. Kiwi で Chrome ウェブストアの Tampermonkey をインストール
  3. Kiwi でこのページの上のボタンをタップ → Tampermonkey の取り込みダイアログで Install
  4. kick.com を開いて 🛡 を確認

Firefox を使う場合: Firefox に Tampermonkey 拡張を有効化して、上のボタンをタップ。

  1. Tampermonkey または Violentmonkey をブラウザにインストール
  2. このページの上のボタンをクリック → 拡張のダイアログで インストール

📱 Kick 公式アプリで見ている方へ

本 Userscript は ブラウザ内でのみ動作します。Kick の公式アプリ（ネイティブアプリ）の中では動きません。

最も簡単な解決策: Safari で kick.com を開いて「ホーム画面に追加」すると、アプリ風に使えて Userscript も動きます（無料・5分）。

機能

  • レイド/ホスト時の自動リダイレクトを検知してブロック
  • 3 モード: すべてブロック / 許可リスト / ブロックリスト
  • ブロック時のトースト通知 + 履歴（直近 50 件）
  • kick.com 右下の 🛡 ボタンから設定（拡張メニュー不要なので iPhone でも操作可）
  • クライアントローカル動作のみ・外部送信なし

仕組み

「ユーザー操作なしに kick.com/A から kick.com/B に切り替わろうとしたら止める」シンプルな実装。

SPA ルーター・Location.assign/replace・location.href セッター・合成クリックの 4 層をフックします。 通常のサイト操作（ブラウズ・カテゴリ・VOD 等への遷移、自分のクリック）は影響しません。

🔒 セキュリティ・プライバシー

外部送信なし · kick.com 限定 · ローカル保存のみ — 一切のテレメトリ・アナリティクス・トラッキングを送信しません（CIで毎push自動チェック）。

  • 禁止API（fetch / XMLHttpRequest / WebSocket / GM.xmlHttpRequest / eval）の使用ゼロ
  • Cookie・パスワード・支払い情報には一切アクセスしない
  • ストレージから読んだ設定は sanitizeConfig() で型・regex 検証
  • slug: [a-z0-9_-]{1,32} のみ / リスト最大500件 / ログ最大50件
